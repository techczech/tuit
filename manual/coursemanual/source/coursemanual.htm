Elements of Online Course: Teacher View

Compose exercises

The key section of the course. Contains tools for the creation of language exercises. See below for details.

Compose sets

Combine exercises into sets. A set can consist of any number of exercises. Very commonly only one.

Exercises only become available for students when they are combined into sets.

Correct tests

Contains list of students on the system and under each student is a list of new (uncorrected) exercises the student has completed. The teacher corrects the exercises and adds comments. The exercises are pre-corrected by the computer depending on how the exercise was defined.

View prepared sets

Preview sets created under Compose sets. Teachers can also try to complete the set and correct it through Correct tests to see whether it works as intended.

Make skins

Skins are templates which define the look of the system. Elements which can be defined are Background color of different page components, Text color, Text size and Font type.

File manager

Tool for deleting sets, exercises and skins.

See the results

Review all the past results of each student. Add general notes or messages to students’ accounts. If you tick Public the message will go to the student, if not, it will only be shared with other teachers.

Change your user info

Change your email address, whether you receive notification, and your skin. Also set yourself up as the default teacher for certain students.

Process user accounts

Set up new accounts for new students or new teachers.

Process account requests

Approve or reject requests for an account made by students.

Types of exercises: Composing guides

Questions and answers

The simplest exercise type. Teachers first decide how many questions the exercise contains. Then they type questions for students to answer. If the type of exercises allows, they can suggest correct exercises for the computer to correct automatically.

Choosing alternatives

The most reliable exercise for correction by computer. It is basically a multiple choice exercise. The teacher selects how many questions there will be and how many answers students will choose from. Then questions and options are typed in and correct option for each sentence chosen. Options can be identical for all questions or there can be different options for each question. The options are displayed in a column and students choose by pressing radial categories.

Matching sentences

Another type of multiple choice exercise which can be best used for matching or ordering sentences. The teacher select how many sentences there will be to match and then types in the answers (or if the task is to put them in the correct order, the sentence which belongs in the position. The student will then have to choose from a pull down menu which sentence should be in that position.

Fill in the blank
(cloze)

The most useful and versatile type. All the other exercises could be recreated using it. It allows very complicated formatting using HTML. Teachers enter or paste text in a large text field and mark blanks for students using [ ] and different parameters. These blanks are displayed for students as empty fields or pull down menus. See below for details of how to mark up exercises.

Crossword Puzzles

Tool for creating simple (Czech-type) crossword puzzles with a tajenka where words are filled across and tajenka will appear in one of the columns. The teacher needs to first plot the crossword in a table and determine the total number or columns and rows as well as the column which will be the tajenka. Fill in the Blank Language

The can include all the following in your exercises:

  1. Pull down menu with one correct answer.
  2. Blank field for a word or several words, specify the length, and correct answer.
  3. Blank field for an ending and specify its size.
  4. Blank space for extended text (text area) and specify the size of the text area.

 

1.

Pull down menu

Enclose selected options between [ ]. Separated options by /. Enclose correct option between ( ).

For example, [dobrým/(dobrou)/dobrém] will display as

 and can be clicked to display three optionswithdobroucorrected by the computer as the correct one.

If you want the first option to be empty, put / right after [.

2.

Blank field for words

To insert a default field just put [ ] where you want the field to appear.

To specify the length, put a number between the brackets, e.g. [5] will display a field 5 characters long.

To specify correct answer, write it between ( ).

For example: [6 (Karlem)] will display a blank field six characters long which the computer will accept of correct if the student writes in Karlem.

3.

Blank field for endings

dej to Petr

Use the same procedure as for blank field but precede [ with <nobr>. For example: dej to Petr<nobr>[3(ovi)]will display as

dej to Petr

and will be accepted as correct if student types in ovi.

4.

Text area

To achieve this effect enclose numbers for widths and height for between [ ] separated by comma.

E.g. [60, 10] will display  a text area of 60 characters and 10 lines. Students can put it as much text as they wish or use HTML tags to use formatting (see below).

5.

Correct answers

If the correct answer is a single word, it is enough to precede it with ( rather than enclose it between ( ). E.g. [6(dobrou] will have the same effect as [6(dobrou)].

Quick HTML Reference: Any HTML can be used. Here’s a summary of the most important HTML tags. bold = <B>bold</B>; italics = <I>bold</I>; paragraph = <P>; forced end of line = <BR>; non-breaking space = ‘&nbsp;’; link = <A HREF=”http://www.link.com”>link</A>.

Improving Text Layout - Top 5

This section describes Top 5 ways of using more advanced layout in exercises, such as bold, italics or including links.

These things are achieved using HTML (a language for annotating text so that it can be displayed on the web). HTML uses so called tags to mark the beginning and end of any formatting. Any HTML can be used that would appear between <BODY> and </BODY> tags on a regular web page. Only the most basic HTML is described here.

1.

bold

Enclose any text between <B> and </B>. For example: <B>bold</B> will display as bold.

2.

italics

Enclose any text between <I> and </I>. For example: <I>bold</I> will display as italics.

3.

bold italics

Tags can be combined in anyway. For example, <B><I> bold italics</I></B> will display as bold italics.

4.

end of paragraph

Hitting ENTER is not enough. The web will simply interpret it as a space. To achieve the same effect, you must write in <P> which will offset any text following it into a new paragraph. For example, text text <P> text text will display as:

text text

text text

While

text text ENTER
text text

will display only as: text text text text.

5.

link

To insert a link you must enclose a text that you want to make ‘clickable’ between <A HREF=”http://www.yourlink.com”> and </A>. So for example, <A HREF=”http://www.bohemica.com/czechonine/frontpage.htm”> Czech online </A> will display as Czech online which you can click and get to the front page of Czech online. Any page on the web can be referred to in that way.

Marks the beginning of a table and defines its characteristics.

Marks the beginning of a row.

    <td width="33%">Jsem z</td>

    <td width="33%">Prahy</td>

    <td width="34%">Praha</td>

  </tr>

Marks the end of a row.

Cheating: There’s no need to learn more than the basics of HTML to use it on Czech Online. You can create the table (or any type of HTML) in an HTML editor such as FrontPage and then copy the source into the Czech exercise. Anything between (not including) <BODY> and </BODY> can be used.
